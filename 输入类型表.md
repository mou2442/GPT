输入类型总表｜机器 / 模型 / 人类联合判定版

一、总目标
不是先分清“皮影”还是“绣像”，
而是先厘清：
- 机器（脚本）能稳定处理什么
- 模型（图生视频 / 约束输入）能稳定吃进去什么
- 人类（你）能稳定判断、修订、回收什么

核心原则：
输入材料不是按“好不好看”分，
而是按“是否单纯、是否稳定、是否适合当前环节”分。


二、输入来源类型

1. born-digital
定义：
- 原生数字图
- 数字绘图、数字线稿、透明底素材、导出图层

优点：
- 边界清楚
- 背景可控
- 适合脚本直接处理
- 适合模型直接拿主形

风险：
- 信息可能过少
- 若只有 silhouette，则不适合细节提取

典型用途：
- 皮影动画
- mask
- 分层位移
- 关节拆解


2. scan-based
定义：
- 扫描件
- 平拍纸面，接近正投影
- 有底色但背景较稳定

优点：
- 比实拍干净
- 纹样与结构信息通常比 silhouette 丰富
- 适合做绣像线的中间母本

风险：
- 纸张底色
- 污点
- 页面弯曲
- 需要清理

典型用途：
- 绣像提取
- 结构线提取
- 纹样保留


3. camera-based
定义：
- 手机 / 相机实拍
- 实物、纸面、装置、皮影偶的现场拍摄图

优点：
- 信息丰富
- 保留材质、磨损、光影、现场感
- 适合做母本

风险：
- 透视偏差
- 光照不均
- 阴影干扰
- 背景不纯
- 压缩与锐化痕迹
- 脚本和模型都更容易误判

典型用途：
- 母本提取
- 风格参考
- 后续转中间层，不宜直接图生视频


三、图像格式关注项

1. 文件格式
- JPG / JPEG：常见实拍输入，有损，不适合反复中间处理
- PNG：适合透明底、mask、结构线、中间层
- WEBP：可用，但当前阶段不优先
- 其他：按需处理

2. 压缩方式
- 有损：边缘脏、细节易坏
- 无损：适合中间层保存

3. 背景状态
- transparent：透明底
- solid：纯色底
- paper：纸面背景
- real_scene：真实环境背景

4. 内容级别
- content-rich：有纹样、颜色、细节
- silhouette-only：只有外形、主体为纯黑或单色
- mixed：介于两者之间


四、当前项目里的三类输入角色

A. 直接动画输入
定义：
- 可以直接进入结构包 / 运动表 / 图生视频测试的输入
典型代表：
- 透明底 silhouette PNG

B. 母本提取输入
定义：
- 不直接喂模型，而是先转中间层
典型代表：
- camera-based 实拍图
- scan-based 图

C. 中间层输出
定义：
- 由脚本或人工从母本里提取出的可喂模型、可继续加工的层
典型代表：
- cutout
- silhouette
- lines
- mask
- 分段关节图


五、机器 / 模型 / 人类各自最稳输入类型

1. 机器（脚本）
最稳输入排序：
A. 透明底、单主体、纯黑或单色 silhouette PNG
B. 透明底、主体完整、颜色干净的 PNG
C. scan-based、背景稳定的平面图
D. camera-based 实拍图

原因：
- 机器最怕猜背景、猜阴影、猜纹样
- 输入越单纯，处理越稳

机器稳定能拆出的内容：

a) 对 silhouette-only PNG
可稳定拆出：
- 主体轮廓
- 透明背景
- bbox
- silhouette 层
- mask
- 初级关节分段参考

不稳定或不具备：
- 纹样
- 内部结构线
- 面部细节
- 绣像细节

b) 对 content-rich scan / camera 图
理论可拆出：
- 主体轮廓
- 主体颜色纹样
- 内部结构变化
- cutout / lines / mask

但不稳定来源：
- 阴影误判
- 背景阈值误判
- 花纹与结构混淆
- 拍摄噪声
- 透视偏差


2. 模型（图生视频模型 / 约束输入）
最稳输入排序：
A. 清晰 silhouette / mask / 透明底主体
B. 已分层的 cutout + 结构线 + 运动约束
C. 干净扫描图
D. 原始 camera-based 实拍图直接图生视频

原因：
- 模型最怕语义混杂
- 一张图同时承担“主体 + 背景 + 光照 + 阴影 + 纹样 + 动作理解”时最不稳

模型对不同输入能吃进去的内容：

a) 对透明底 silhouette PNG
结构包里较稳可吃：
- 主轮廓
- 主体整体姿态
- 剪影重心
- 大关节段落
- 独立主体边界

运动表里较稳可吃：
- 整体位移
- 分段旋转
- 摆臂
- 抬头
- 低自由度关节运动
- 局部轻摆

吃不好的：
- 细纹样保持
- 绣像式内部细节
- 丰富材质变化
- 复杂形变

b) 对原始 camera-based 实拍图
可能吃进去一点：
- 大致人物姿态
- 材质氛围
- 颜色调子
- 部分轮廓

容易失真：
- 纹样漂移
- 肢体边缘抖动
- 阴影被当实体
- 背景乱动
- 关节位置不稳定

c) 对 camera-based 转出的中间层
可明显提升：
- 主体边界稳定
- 姿态稳定
- 背景静止
- 局部运动方向
- 风格约束落点

结论：
camera-based 图不适合直接喂模型，
更适合先转中间层。


3. 人类（观察、修订、回收）
人类最稳接收的不是最原始的图，
也不是最自动化的图，
而是“你一眼就知道它该拿来做什么”的图。

a) silhouette-only PNG
人类一眼就知道：
- 是皮影 / 剪影素材
- 适合摆动、旋转、分段
- 用途单纯

b) content-rich camera / scan 图
人类会同时看到：
- 实物
- 材质
- 花纹
- 光照
- 背景
- 风格

优点：
- 信息丰富
- 审美判断空间大

缺点：
- 下一步用途不单一
- 需要你来裁决保留什么、舍弃什么


六、当前样本对照

样本 1 / 样本 2
类型判断：
- source_type: born_digital 或已处理输出
- file_format: png
- background_type: transparent
- content_level: silhouette_only
- preprocessing_route: direct
- best_use: 皮影动画 / mask / 分层位移 / 关节拆解

结论：
- 不是坏图
- 是好素材
- 对脚本、模型、人类都更“单纯”
- 是更优的直接动画输入


样本 3（皮影实拍）
类型判断：
- source_type: camera_based
- file_format: jpg
- background_type: paper / real_scene
- content_level: content-rich
- preprocessing_route: camera_cleanup
- best_use: 绣像提取 / 风格参考 / 母本转中间层

结论：
- 不适合直接喂模型
- 适合作为母本提取输入
- 更适合先转 cutout / silhouette / lines 等中间层


七、中间层三层包的意义

当前三层包定义：
1. cutout
- 保留原始纹样与颜色的透明底主体

2. silhouette
- 纯黑透明底剪影
- 适合皮影线、关节拆解、低自由度运动

3. lines
- 自动提的结构线
- 用于结构参考，不等于最终绣像线稿

工程意义：
- 证明 camera-based 图可转中间层
- 但在“干净有效”这个维度，当前还未超过原始 silhouette PNG


八、当前阶段的优先级判断

对脚本更友好：
透明底单主体 PNG > 实拍图

对模型更友好：
透明底 silhouette / 已分层中间层 > 原始实拍图

对人类更友好：
用途单一、角色明确的输入 > 信息混杂但丰富的输入

综合排序（当前阶段）：
1. 第一二张透明底 silhouette PNG
2. 从第三张实拍图提取出来的中间层
3. 第三张原始实拍图


九、输入优先级规则（当前阶段）

1. 能直接提供透明底 silhouette 的，优先直接作为皮影动画输入
2. camera-based 实拍图，不直接喂模型，先转成中间层
3. 中间层的目标不是“信息最多”，而是“语义最单纯”
4. 结构包和运动表优先服务 silhouette / 分段关节运动
5. 绣像细节保留属于第二阶段，不抢第一阶段的稳定性
6. 原图可收 JPG，但中间层统一尽量输出 PNG
7. 透明底、mask、分层、结构线优先保存为 PNG
8. 输入类型表、内容级别、前处理路线应作为项目字段写入

十一、当前一句话结论

脚本最稳定接收的是透明底单主体图；
模型最稳定接收的是已经降维后的结构化中间层；
人类最稳定接收的是用途单一、角色明确的输入。

因此当前阶段：
- 第一二张 PNG 是更优的直接动画原材料
- 第三张实拍图是更优的母本提取原材料
- 实拍图转中间层有价值，但目标应是“语义单纯”，不是“信息最多”
