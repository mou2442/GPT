待回收结论 A｜模型作为低精度分镜约束执行器 + 路线A（协同创作）

一、当前实操结论
当目标是“明确设计、精准体现”时，
封装生成平台不是严格执行器，
而是高波动解释器。

模型更像在做：
- 低精度分镜补全
- 帧级连续化
- 像素级生成
而不是：
- 严格动作执行
- 严格关系调度
- 严格原画还原

二、分层理解
人类：
- 语义设计
- 关系设计
- 分镜意图
- 锁定项与禁止项

结构包：
- 对象级
- 层级级
- 锁定项
- 可动 / 不可动对象

动作表：
- 秒段级
- 部件级
- 方向
- 幅度
- 节奏
- hold / settle / forbid

模型：
- 帧级推演
- 像素级落实
- 自动补中间态
- 自动解释未明说部分

压缩表达：
人类 = 镜头语义级
结构包 = 对象与层级级
动作表 = 秒段与部件级
模型 = 帧级连续化
结果 = 像素级生成

三、实操暴露出的核心问题
平台封装模型在“精准体现”任务里，
主要风险不在画质，而在结构执行的不确定性：

1. 守位不稳
- 相对位置漂移
- 构图重心偏移
- 角色之间的空间关系松动

2. 守线不稳
- 线稿风格漂移
- 黑白线描长成厚涂或立体感
- silhouette / 白描无法稳定保持

3. 守关系不稳
- 谁先动、谁回应变模糊
- 试探 / 拒绝 / 恳求等关系逻辑被稀释
- 模型只给“像在互动”的近似结果

四、对 prompt 的现实判断
自然语言提示词不是直接写视频，
而是在写一份低精度分镜约束。

纯文学描述：
- gently
- subtly
- elegant
- restrained
对审美有用，
对控制较弱，
常导致“几乎不动”或“自由脑补”。

更有效的是：
- 对象明确
- 部件明确
- 秒段明确
- 动作方向明确
- 幅度明确
- 禁止项明确

五、动作表应推进到的粒度
动作表 v1 字段建议：
- object_id
- part
- time_start
- time_end
- action
- direction
- amplitude
- speed
- hold / settle
- lock / forbid

示例：
A1｜跪坐人物｜head｜0.0｜1.2｜lift｜up｜small｜slow｜no｜background locked
A2｜跪坐人物｜hands｜0.6｜2.0｜move｜forward｜small｜slow｜yes｜no large deformation
B1｜右侧女子｜head｜1.8｜3.0｜tilt｜down-left｜very small｜slow｜yes｜body mostly locked
B2｜右侧女子｜torso｜2.0｜3.2｜shift｜back｜very small｜slow｜yes｜chair locked

六、路线A：协同创作
定义：
接受模型会脑补、会近似实现、会带来随机性。
人类更像导演、筛片人、误差回收者。

适合：
- 灵感探索
- 风格探测
- 快速试片
- 意外结果收集
- 低约束活化
- 找“可能的感觉”

工作链：
结构包 + 动作表
→ prompt 压缩
→ 平台生成试片
→ 看片回收误差
→ 判断哪类约束常失效
→ 再决定是否继续用同平台 / 同模型

平台在路线A里的角色：
- 试片器
- 气质猜测器
- 风格近似器
- 低精度分镜补全器

不应把它当成：
- 精确分镜执行器
- 严格动作还原器
- 高保真关系调度器

七、对项目的现实定位修正
“结构包 + 动作表 → prompt → 直接出成片”
这个想定不稳。

更现实的写法应是：
结构包 + 动作表
→ 测试性 prompt
→ 平台生成试片
→ 回收误差
→ 反推失效控制点
→ 判断是否进入下一轮协同创作

一句话结论：
平台生成在当前阶段更适合作为“低精度分镜试片层”，
而不是“精准设计的最终执行层”。
